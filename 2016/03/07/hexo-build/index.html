#本站起源————前端新手从零开始使用hexo+coding搭建静态博客

##为什么是hexo与coding？

  对于一个前端小白来说，在wordpress、typecho等各种主流的博客模板中，选择hexo完全是因为这是一款基于Node.js的静态博客框架，可以在没有繁琐复杂的后台和数据库情况下以生成静态文件的方式完成一个简单小型博客的搭建。

  至于coding，是因为hexo其实本地就可以直接用，但是如果想展示给别人或者自己看起来方便，这自然就需要部署到线上。而github就刚好为我们提供了这样的一个平台，它可以将hexo生成的静态网页直接在线托管，我们只要在本地写好后利用几个简单的命令就可以生成部署好。由于博主英语渣，而且极度不习惯github的操作方式，所以就用了中文版github———coding，当然还是推荐大家学习和使用github。^_^

PS : 强烈建议，如果你对前端略有了解，有一定计算机基础，愿意折腾，那么hexo将非常适合你。但是如果你只是想拥有一个简单易写的博客，个人感觉在线的平台会更加适合你，这里推荐简书(如果能早发现，博主肯定不会再这么折腾了)。

但是coding的话还需要一些简单的操作：

1.进入你的项目中的演示模式的配置页面里。

2.在开启演示模式之前，会提醒你没有检测到环境，直接强制开启就可以了。

3.接着在高级选项中把运行环境设为 HTML ，部署版本为 master ，设置访问域名。然后应用内存选用 256M 就足够了。

4.最后点击一键部署，等待部署成功即可，然后你可以访问域名来查看自己是否部署成功。

####自动部署

每次我们在本地写完文章，生成静态页面后push到仓库里，就需要重新部署一次，不然我们是看不到新的文章出现在自己的博客里。但是每次都要繁琐的部署也挺浪费时间的。所以我们就需要用到webhook来帮我们自动部署。

1.在设置里找到webhook,然后点击新建Hook，

第一个输入框中是填你的博客域名，也就是前面部署的时候那个域名地址，然后在后面加上/_。

第二个输入框是输入一个token，我们直接填写就可以了。

2.在演示平台中找到环境变量，新建一个变量，变量名填写为WEBHOOK_TOKEN，值为，接着重新启动应用就 ok 了。
